PaGE_5.1 Documentation - the --num_permutations command

The PaGE confidence measure is permutation based. This option changes the default number of permutations.

The 2-sample unpaired case. When there are two groups and the data is unpaired, and there aren't too many replicates, then PaGE uses all ways of switching the columns of data between the two groups. If the groups have size n and m then there are n+m choose n permutations.

The direct comparison case. In the direct comparison case, unlogged data, the data consist of ratios. A permutation consists of taking a subset of these ratios and forming their reciprocals. If there are n replicates, then there are 2n such permutations. If working with the logged data, a permutation consits of taking a subest of the M values and multiplying them by negative one.

The paired case. In the paired case, if unlogged data, pairwise ratios are formed, or if logged data then pairwise differences are formed. The data are thus reduced to the direct comparison case.

Note: If there are more than 200 permutations then PaGE will just use 200 random permutations. If you want to raise the default number above 200 (for greater accuracy), or below 200 (for greater speed), then use the --num_permutations command.

This command takes one argument of type integer, which must be greater than 2.

Example: 

> perl PaGE_5.1.6.pl --num_permutations 500
