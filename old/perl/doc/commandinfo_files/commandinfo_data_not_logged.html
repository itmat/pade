PaGE_5.1 Documentation - the --data_not_logged command

Use this option to indicate that the date file contains unlogged data.

This command takes no arguments.

Note 1: In the case of 2-channel data then this indicates the data file log ratios.

Note 2: If your data is unlogged, but contains negative values, then there are restrictions on the kinds of analyses you can perform. For example, if there are negative intensities, then any analysis based on fold change (i.e. ratios) is meaningless. If any such analysis is requested, you will first be asked to shift the data to be all positive. For more on this see --shift and --means.

Example: 

> perl PaGE_5.1.6.pl --data_not_logged
